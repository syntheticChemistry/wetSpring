# wetSpring Provenance Braids

Ferment transcript braids for lithoSpore downstream consumption.

These are the **portable provenance artifacts** — small JSON files that travel
with the code (repo, USB, guideStone). The raw data (FASTQs, BAMs, breseq
output) stays on NUCLEUS storage. The braid proves the computation happened
and records what was computed, by whom, on what substrate, and when.

## Chain Model

```
NUCLEUS (4TB NVMe)                    Repo / USB / guideStone
├── 200 GB raw reads                  ├── braids/barrick_2009_sovereign.json  (~1 KB)
├── breseq output (GBs)         →     ├── braids/barrick_2009_breseq.json     (~1 KB)
├── sovereign pipeline output         └── braids/tenaillon_2016_sovereign.json (~1 KB, pending)
└── full provenance DAG sessions
```

The spore can't carry the mountain. But it proves the mountain was climbed.

## Wire Format

All braids follow the lithoSpore ferment transcript contract
(see `infra/wateringHole/handoffs/LITHOSPORE_FERMENT_TRANSCRIPT_BRAID_HANDOFF_MAY17_2026.md`):

```json
{
  "dataset_id": "...",
  "spring": "wetSpring",
  "braid_id": "braid-...",
  "dag_session_id": "dag-...",
  "computation": {
    "tool": "...",
    "substrate": "...",
    "input_accession": "...",
    "output_blake3": "..."
  }
}
```

## For lithoSpore

1. Pull these braids into `data.toml` as `upstream_braid` entries
2. The guideStone validates the braid metadata (accession, hash) without
   needing the raw data
3. When connected to NUCLEUS, the full DAG session can be resolved for
   deep audit
4. Multiple braids for the same dataset (breseq vs sovereign) show the
   elevation from external C++ baseline to sovereign Rust+GPU composition

## Regeneration

Braids are regenerated by the sovereign pipeline binary and overwrite
the previous version. The braid includes a timestamp and hash so each
generation is distinguishable.

```bash
cargo run --features gpu --bin validate_sovereign_resequencing --release
```
